# DB-WORK — مزامنة سجل السائقين من أبشر/تم إلى قاعدة البيانات

أدوات تأخذ ملف Excel المُصدَّر من **أبشر / تم** (قائمة المركبات) وتحدّث **سجل السائقين**
في قاعدة البيانات بشكل آلي وآمن.

## الحقول
| المطلوب | مصدره في ملف أبشر |
|---|---|
| رقم اللوحة | عمود «رقم اللوحة» |
| نوع المركبة | «الماركة» + «الطراز» + «سنة الصنع» |
| رقم الإقامة | عمود «رقم هوية المستخدم الفعلي» |
| اسم الموظف | عمود «اسم المستخدم الفعلي» |
| **رقم الجوال** | **غير موجود في أبشر** — يُحفظ من قاعدة البيانات (أو من تصدير «تم» لاحقاً) |

المطابقة بين الإكسل وقاعدة البيانات تتم عبر **رقم الإقامة** (المفتاح الأثبت).

## الملفات
- `absher_parser.py` — يقرأ ملف أبشر ويكتشف الأعمدة بالاسم (يعمل حتى لو تغيّر ترتيبها).
- `sync_drivers.py` — الأداة الرئيسية: يحسب الفروقات ويطبّقها.
- `out/` — مخرجات: `sync_report.txt`، `proposed_changes.json`، `deauthorized.json`.
- `backups/` — نسخة احتياطية كاملة لجدول `drivers` تُؤخذ تلقائياً قبل أي تطبيق فعلي.

## التصنيفات التي يحسبها
- **➕ إضافة:** مستخدم فعلي في أبشر غير موجود بقاعدة البيانات → يُضاف (الجوال فارغ).
- **✏️ تحديث:** موجود في الاثنين لكن تغيّرت لوحته/مركبته → تُحدّث (الجوال والرقم الوظيفي والاسم العربي تُحفظ).
- **🚫 تم إلغاء تفويضهم:** سائق له لوحة في قاعدة البيانات ولم يعد المستخدم الفعلي لأي مركبة في تقرير أبشر.
- **🅿️ بلا مركبة:** سائق في قاعدة البيانات بلا لوحة (لا يُعتبر ملغى تفويضه آلياً).

## الاستخدام

### 1) معاينة بدون اتصال (آمنة تماماً — لا تعدّل شيئاً)
تستخدم `drivers_data.js` كصورة لقاعدة البيانات:
```bash
python DB-WORK/sync_drivers.py \
  --from-json drivers_data.js \
  --excel "اخر تحديث لقائمة المركبات اللواري والدينات_2026-06-28.xlsx" \
          "اخر تحديث لمركبات السيارات الصغيرة سيدان_2026-06-28.xlsx"
```
راجع `DB-WORK/out/sync_report.txt`.

### 2) معاينة من قاعدة البيانات الحيّة (على الخادم 509.rip)
```bash
export DATABASE_URL="postgres://...";  pip install psycopg2-binary openpyxl
python DB-WORK/sync_drivers.py --from-db --excel <ملفات أبشر>
```

### 3) التطبيق الفعلي (على الخادم فقط، بعد مراجعة التقرير)
```bash
python DB-WORK/sync_drivers.py --from-db --excel <ملفات أبشر> --apply
```
- يأخذ **نسخة احتياطية** في `DB-WORK/backups/` قبل أي كتابة.
- يحدّث جدول `drivers` (لوحة/مركبة، ويُبقي الجوال والاسم العربي).
- يكتب قائمة «تم إلغاء تفويضهم» في بلوب `deauthorized_data` فتظهر تلقائياً أسفل الجدول الأسبوعي في الموقع.
- خيار `--update-names` لاستبدال الأسماء بأسماء أبشر (افتراضياً تُحفظ الأسماء الحالية).

## الأمان
- الافتراضي **تجريبي** (dry-run). لا تعديل إلا مع `--apply` و `--from-db` معاً.
- نسخة احتياطية كاملة قبل كل تطبيق → يمكن الرجوع عبر استعادة `drivers_backup_*.json`.
- الجوال لا يُمسّ أبداً (غير موجود في أبشر).
