## 1) نبذة عن المشروع
نظام إدارة أسطول لـ **«شركة بن زومة للتجارة والتطوير الدولية» — فرع الدمام، قسم الحركة**.
- **الباك إند:** Flask (Python 3.12)، gunicorn، **PostgreSQL** عبر `psycopg2` مع **fallback إلى SQLite**.
- **الفرونت إند:** Vanilla JS + قوالب Jinja (RTL عربي)، تصدير Excel عميل-جانبي (ExcelJS + FileSaver)، PDF (html2pdf.js)، خرائط (Leaflet للـGPS).
- **متحكّم مشترك:** `static/app_ux.js` + نظام تصميم `static/base_styles.css`.
- **هوية:** كحلي `#0C2340` + ذهبي `#c9a227`.

### طبقة قاعدة البيانات (مهم)
- صفّ JSON واحد لكل تبويب، بـ `id=1` للموقع الأساسي و **`id=2` لصندوق محطة العمل (sandbox)**.
- الدوال: `blob_get/blob_set` و`_ws_get2/_ws_put2`، مع `_row_id()` و`is_workstation()`.
- بعد إصلاحات هذه الجلسة: تُمرَّر أسماء الجداول عبر `_safe_tbl()` (تحقّق هوية) و`_loads_blob()` (try/except على JSON تالف).

### عزل محطة العمل (Workstation) — قاعدة صارمة
- مسار منفصل: `/importantworkstation/*` (بادئة URL). `is_workstation()` يعتمد على `request.path`.
- التبويبات المقفلة بكلمة سر: الموظفون، مزامنة GPS، الكاميرات، **التتبع**. كلمة السر: `Kn-123123` (اختارها المالك عمداً).
- **الموقع الأساسي `/` يجب ألا يتأثر إطلاقاً** بتعديلات محطة العمل (لا بيانات، ولا localStorage — بادئة `ws:`، ولا قوالب). أي تسريب = خطأ حرج.

### الكسوة المؤسسية (Enterprise Shell)
- ميزة مُلتزَمة في commit `636613d`: وضع داكن عميق + **قائمة جانبية ثابتة على اليمين** + شريط علوي مطوّر.
- تُبنى في `app_ux.js` → `buildEnterpriseShell()` على أي صفحة فيها `.bz-topbar` (تتجاهل `/dashboard` لها كسوتها الخاصة `body.bz-dash`).
- مُنسّقة في `base_styles.css` تحت `body.bz-enterprise`. **القائمة مثبّتة فيزيائياً على اليمين** (`right:0`/`padding-right`، commit `bcf403f`) — لا تُرجِعها لـ`inset-inline-*`.
- نقاط نهاية تدعمها: `/api/whoami`، `/api/expiry_alerts_preview`.

### كسر الكلمات في رؤوس الجداول
- `.nb { white-space: nowrap }` في base_styles.css. كل كلمة في الترويسة تُلَفّ بـ`<span class="nb">` فلا تنكسر الكلمة المفردة (مثل «الموديل/الحمولة») في منتصفها.

5. **اكتشاف الأخطاء:** المالك يريد أن **تُبلّغه بالأخطاء المكتشفة وتأخذ موافقته قبل الإصلاح** (لا تُصلِح بصمت).
6. عند `git`: افحص التفرّع قبل الرفع؛ لا force-push بدون موافقة + نسخة احتياطية فرع.
7. لا تعدّل منطق تصدير Excel/PDF عند تغيير الشكل (المالك راضٍ عنه).
