# نظام إدارة أسطول بن زومة (BIN ZOMAH INTL. Fleet Management)

نظام سحابي متكامل لإدارة الأساطيل، مُصمم لمتابعة عمليات وجداول وعهد السائقين بدقة ومرونة عالية. يتميز النظام بمعمارية هجينة تدعم الاستضافة المحلية والسحابية مع نظام تخزين Blob لضمان أقصى درجات المرونة.

## 🌟 الميزات الأساسية

1. **لوحة القيادة الذكية (Executive Dashboard):**
   - متابعة حية لعدد السائقين النشطين وغير المجدولين.
   - تنبيهات تلقائية للوثائق المنتهية أو القريبة من الانتهاء (إقامات، رخص).
   - إحصائيات بيانية لنسب التوزيع والترابط بأنظمة الموارد البشرية.

2. **سجل السائقين والمركبات (Registries):**
   - إدارة السائقين مع دعم الربط بأنظمة (أبشر) عبر الإقامات.
   - تتبع المركبات، اللوحات، بطاقات التشغيل، وبيانات الاتصال.

3. **العمليات والجدولة (Operations):**
   - جدول أسبوعي لتوزيع السائقين على مسارات العمل.
   - نقل عهدة المركبات بين السائقين بشكل فوري وتوثيقها رسمياً.
   - جدول متخصص لغسيل السيارات الدورية.

4. **الصيانة والورشة (Maintenance):**
   - توثيق دخول وخروج المركبات للورشة.
   - متابعة تغيير الزيوت والمشتريات التشغيلية للأسطول.

5. **سجل الحوادث والتوثيق (Incidents):**
   - توثيق الحوادث وتاريخها والمسؤول عنها.
   - سجل عهد متكامل (Handover) يشمل تفاصيل حالة المركبة وقت التسليم.

## ⚙️ البنية المعمارية (Architecture)

- **Backend:** `Python 3.12` with `Flask`.
- **Database (Hybrid):**
  - **SQLite:** للتشغيل المحلي وأخذ النسخ الاحتياطية.
  - **PostgreSQL:** للعمل على الاستضافات السحابية.
- **Storage Pattern:** يعتمد النظام بشكل أساسي على `Blob Storage`، حيث يتم تحويل بيانات الأقسام إلى `JSON` مضغوط وحفظها في نصوص `TEXT` لضمان التوافق بين `SQLite` و `PostgreSQL` بدون الحاجة إلى `Migrations` معقدة.
- **Frontend:** `HTML5`, `CSS3`, `Vanilla JS`, `Chart.js`.

## 🚀 كيفية التشغيل والتثبيت (Deployment & Setup)

### 1. التشغيل المحلي (Local Development)
1. قم بإنشاء بيئة وهمية (Virtual Environment):
   ```bash
   python -m venv venv
   source venv/bin/activate  # (Linux/Mac)
   venv\Scripts\activate     # (Windows)
   ```
2. تثبيت الحزم المطلوبة:
   ```bash
   pip install -r requirements.txt
   ```
3. تشغيل النظام:
   ```bash
   python app.py
   ```
   *سيعمل النظام على `http://localhost:3000` وسيقوم تلقائياً بإنشاء قاعدة بيانات `database.sqlite`.*

### 2. التشغيل على الاستضافة السحابية (Production / Docker)
يحتوي المشروع على ملف `Dockerfile` و `Procfile` جاهزين لدعم منصات مثل Railway أو Render.
يتم تشغيل النظام باستخدام `Gunicorn`:
```bash
gunicorn --workers 1 --threads 8 --bind 0.0.0.0:3000 app:app
```

## 🔒 الصلاحيات والأمان
- **Admin:** يمتلك الصلاحيات الكاملة للقراءة والكتابة، ونقل العهد، ومشاهدة سجل التدقيق (`Audit Log`).
- **User:** صلاحية مخصصة للمشاهدة أو التعديل المحدود.
- **الفروع (Multi-Branch):** الإدارة العليا تمتلك حق رؤية كافة الفروع (HQ)، بينما يتم حصر مدراء الفروع برؤية بيانات فرعهم فقط.
